# Session Summary - February 27, 2026

## Overview

Major debugging session focused on fixing NIP-46 remote signer authentication and file upload issues.

## Issues Fixed

### 1. Self-Hosted Crypto Libraries
**Problem**: CDN libraries (libsodium, noble-curves, Yjs) were being blocked due to MIME type mismatches (`text/plain` instead of `application/javascript`).

**Solution**: Self-hosted all crypto libraries in `web/js/vendor/`:
- `libsodium.js` (~521KB) - Core libsodium module
- `sodium.js` (~102KB) - libsodium-wrappers
- `noble-secp256k1.min.js` (~40KB) - Bundled noble-curves
- `yjs.min.js` (~92KB) - Bundled Yjs with dependencies

**Commits**: `335de2b`, `2f77edf`, `4ad6be8`

### 2. Upload Metadata Publishing
**Problem**: Uploads showed "Success" but files disappeared. The metadata publish to the relay was silently failing.

**Solution**:
- Made `NIP46.publishEvent()` wait for OK responses instead of fire-and-forget
- Removed try-catch that was swallowing publish errors
- Upload now properly fails if metadata can't be stored

**Commits**: `d34f5f8`, `103a039`

### 3. NIP-42 Auth Identity Mismatch
**Problem**: Relay rejected published events with "restricted: you can only publish as your authenticated identity". The NIP-42 auth used the ephemeral client keypair, but events were signed with the user's actual pubkey.

**Solution**: Implemented lazy re-authentication:
- Login proceeds without extra auth overhead
- On first "restricted" error, trigger re-auth with user's pubkey
- Remote signer signs NIP-42 AUTH event
- Retry pending events after re-auth succeeds

**Commits**: `b0fd2a7`, `2276d81`, `33525ab`

### 4. Login Timeout
**Problem**: Login was timing out because proactive re-auth created race conditions with other signing requests.

**Solution**: Removed proactive re-auth, switched to lazy re-auth on "restricted" errors only.

**Commit**: `33525ab`

### 5. Rate Limiting
**Problem**: Bulk uploads hit relay rate limits ("you are noting too much").

**Solution**: Added 500ms delay between uploads in bulk operations.

**Commit**: `4ad6be8`

### 6. UI Fixes
- Fixed NIP-46 modal staying open after session restore (`b6925a0`)
- Added login progress indicators (`e3b94dc`)
- Fixed `Auth.queryShares` not found error (`103a039`)

### 7. CI/CD Fixes
